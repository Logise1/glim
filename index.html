Capítulo 1: ¡Hola, Glim!

Bienvenido al mundo de Glim. Glim es un lenguaje de programación diseñado para ser simple, visual y divertido. En lugar de escribir programas complejos, en Glim das órdenes directas para crear y manipular objetos en un escenario 2D.

El Escenario y los Objetos

Imagina que tienes un lienzo en blanco llamado escenario. En este escenario, puedes crear objetos. El comando más fundamental es init, que crea un nuevo objeto.

Vamos a crear nuestro primer objeto, una bola. Copia este código en el editor de Glim y pulsa "Ejecutar":

init bola

¡Felicidades! Has creado tu primer objeto. Por defecto, es un círculo blanco en el centro del escenario (coordenadas x=0, y=0).

¡Ponte a prueba!

Crea dos objetos diferentes: uno llamado jugador y otro llamado enemigo.

Mostrar Solución

init enemigo

Capítulo 2: Objetos y Propiedades

Cada objeto en Glim tiene propiedades que puedes cambiar, como su posición, tamaño y color. Para modificar una propiedad, usamos el comando set ... to ... o el atajo con el signo igual =.

Propiedades Básicas

  • x, y: La posición horizontal y vertical del objeto. El centro (0,0) está en medio del escenario.
  • size: El tamaño del objeto (su radio si es un círculo).
  • color: El color del objeto. Puedes usar nombres de colores en inglés ("red") o códigos hexadecimales ("#ff0000").

Vamos a mover nuestra bola, hacerla más grande y cambiarle el color.

init bola

# Mover la bola a la derecha y hacia arriba
bola.x = 100
set bola.y to 50

# Cambiar su tamaño y color
bola.size = 30
bola.color = "#38bdf8" # Un bonito color cian

¡Ponte a prueba!

Crea un objeto llamado sol en la esquina superior izquierda del escenario. Haz que sea grande y de color amarillo.

Mostrar Solución 
init sol
sol.x = -250
sol.y = 150
sol.size = 50
sol.color = "yellow"

Capítulo 3: Movimiento y Lógica

La magia de los juegos y animaciones ocurre cuando las cosas cambian con el tiempo. Para ello, necesitamos variables y lógica condicional.

Variables

Una variable es un contenedor para guardar un número o un texto. Las creamos con set o =.

set velocidad to 5

Ahora podemos usar la variable velocidad en nuestros cálculos. Por ejemplo, para mover un objeto:

jugador.x = jugador.x + velocidad

Lógica Condicional con `if`

El comando if nos permite ejecutar código solo si una condición es verdadera. Es la base para tomar decisiones.

Por ejemplo, podemos hacer que un objeto rebote al llegar al borde derecho del escenario.

if jugador.x > 320
  set velocidad to -5 # Invertimos la velocidad

¡Ponte a prueba!

Crea una variable puntuacion con valor inicial 0. Luego, usa un if para que, si un objeto moneda está en la posición x=0, la puntuación aumente en 10. Usa el comando log para ver el resultado en la consola.

Mostrar Solución 
init moneda
moneda.x = 0

set puntuacion to 0

if moneda.x = 0
  puntuacion = puntuacion + 10

log puntuacion # Debería mostrar 10 en la consola

Capítulo 4: Interactividad

¡Hagamos que nuestros proyectos respondan al usuario! Glim puede detectar pulsaciones de teclado y la posición y clics del ratón.

Entrada de Teclado

Usamos una condición if especial para detectar si una tecla está siendo presionada.

# Mover al jugador a la izquierda con la flecha
if key flechaizquierda pressed
  jugador.x = jugador.x - 5

Puedes usar flechaarriba, flechaabajo, flechaizquierda, flechaderecha, espacio o cualquier letra (a, b, w, etc.).

Entrada del Ratón

Glim guarda automáticamente la posición del ratón en dos variables: mouse_x y mouse_y. También puedes detectar clics:

# Un objeto 'mira' sigue al cursor
set mira.x to mouse_x
set mira.y to mouse_y

# Teletransportar al jugador al hacer clic izquierdo
if mouse izquierdo pressed
  jugador.x = mouse_x
  jugador.y = mouse_y

Puedes usar izquierdo, medio o derecho para los botones del ratón.

¡Ponte a prueba!

Haz que el jugador salte (aumente su `y`) cuando se presione la barra espaciadora.

Mostrar Solución 
if key espacio pressed
  jugador.y = jugador.y + 10

Capítulo 5: Bucles de Juego

Hasta ahora, nuestro código se ejecuta una sola vez de arriba a abajo. Para crear movimiento y animación, necesitamos que el código se repita una y otra vez. Esto se conoce como el bucle de juego y lo creamos con while.

Un bucle while repetirá el código que está dentro mientras su condición sea verdadera. Para un bucle infinito, usamos una condición que siempre es cierta, como 1 = 1.

init bola
set velocidad to 2

while 1 = 1
  # Este código se repetirá 60 veces por segundo
  bola.x = bola.x + velocidad

  if bola.x > 320 or bola.x < -320
    velocidad = velocidad * -1
  
  # Es importante añadir una pequeña pausa para que el
  # navegador pueda refrescar la pantalla.
  sleep 0.016 
endwhile

¡Casi todo tu código de juego estará dentro de un bucle while!

¡Ponte a prueba!

Crea un objeto que se mueva en diagonal por la pantalla y rebote en los cuatro bordes.

Mostrar Solución 
init bola
set vel_x to 3
set vel_y to 2

while 1 = 1
  bola.x = bola.x + vel_x
  bola.y = bola.y + vel_y

  if bola.x > 320 or bola.x < -320
    vel_x = vel_x * -1

  if bola.y > 180 or bola.y < -180
    vel_y = vel_y * -1
  
  sleep 0.016
endwhile

Capítulo 6: Disfraces y Sonidos

Puedes personalizar tus objetos con imágenes (llamadas disfraces) y añadir efectos de sonido para dar vida a tus creaciones.

Gestor de Disfraces y Sonidos

En el editor de Glim, haz clic en los botones "Disfraces" y "Sonidos". Se abrirán ventanas donde puedes subir tus propios archivos de imagen y audio. Es importante que los nombres que les des no tengan espacios ni caracteres especiales.

Usando Disfraces

Una vez que has subido una imagen, puedes asignarla a un objeto con la propiedad costume. ¡Recuerda que el nombre del disfraz debe ir entre comillas!

init nave
nave.costume = "mi_nave_espacial"

Reproduciendo Sonidos

Para reproducir un sonido que has subido, usa el comando play sound.

if key espacio pressed
  play sound "disparo_laser"

¡Ponte a prueba!

Sube una imagen cualquiera y asígnala a un objeto. Luego, sube un archivo de sonido corto y haz que se reproduzca cuando haces clic con el ratón.

Mostrar Solución 
init personaje
personaje.costume = "nombre_de_tu_imagen"

while 1 = 1
  if mouse izquierdo pressed
    play sound "nombre_de_tu_sonido"
  
  sleep 0.016

Capítulo 7: Proyecto Final - Esquiva los Bloques

¡Es hora de juntar todo lo que hemos aprendido! Vamos a crear un juego simple donde controlas a un jugador que debe esquivar bloques que caen.

Paso 1: El Jugador

Crea el jugador y haz que se mueva de izquierda a derecha con las flechas.

init jugador
jugador.y = -150
jugador.color = "cyan"
jugador.size = 20

while 1 = 1
  if key flechaizquierda pressed
    jugador.x = jugador.x - 5
  if key flechaderecha pressed
    jugador.x = jugador.x + 5
  
  sleep 0.016
endwhile

Paso 2: El Enemigo que Cae

Ahora, añade un enemigo que caiga desde la parte superior. Cuando llegue al fondo, debería reaparecer arriba en una posición horizontal aleatoria.

init enemigo
enemigo.color = "tomato"
enemigo.size = 25
enemigo.y = 200

# Dentro del bucle while, después del código del jugador:
enemigo.y = enemigo.y - 4 # Que caiga

# Si el enemigo se sale por abajo...
if enemigo.y < -200
  # ...lo reiniciamos arriba
  enemigo.y = 200
  # ...y en una 'x' aleatoria
  set enemigo.x to random from -300 to 300

Paso 3: Detección de Colisión

El juego termina si el jugador toca al enemigo. Usamos la condición touches. Cuando colisionan, podemos parar el juego con el comando stop.

# Dentro del bucle while, al final:
if object jugador touches enemigo
  log "¡Has perdido!"
  stop # Detiene la ejecución del juego

Código Completo del Juego

Mostrar Código Completo 
# -- CONFIGURACIÓN INICIAL --
init jugador
jugador.y = -150
jugador.color = "cyan"
jugador.size = 20

init enemigo
enemigo.color = "tomato"
enemigo.size = 25
enemigo.y = 200
set enemigo.x to random from -300 to 300

# -- BUCLE PRINCIPAL DEL JUEGO --
while 1 = 1
  # -- LÓGICA DEL JUGADOR --
  if key flechaizquierda pressed
    jugador.x = jugador.x - 5
  if key flechaderecha pressed
    jugador.x = jugador.x + 5
  
  # -- LÓGICA DEL ENEMIGO --
  enemigo.y = enemigo.y - 4 # El enemigo cae

  # Si el enemigo se sale por abajo, reiniciarlo
  if enemigo.y < -200
    enemigo.y = 200
    set enemigo.x to random from -300 to 300
  
  # -- LÓGICA DE COLISIÓN --
  if object jugador touches enemigo
    log "¡HAS PERDIDO!"
    stop # Detiene el juego
  
  # Pausa para el refresco de pantalla
  sleep 0.016
endwhile
